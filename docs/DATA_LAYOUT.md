# Data Layout

The release bundle expects data to be mounted under `open_source_release/data/` by default. You can override every path with CLI flags or environment variables.

## Vimeo90K

```text
data/vimeo_triplet/
  sequences/
    00001/
      0001/
        im1.png
        im2.png
        im3.png
  tri_trainlist.txt
  tri_testlist.txt
  mv_cache/
    00001/
      0001.npz
  prealigned_v2/
    00001/
      0001_im1.png
      0001_im3.png
  motion_labels.csv
```

Notes:

- `mv_cache/` is the output of MV extraction.
- `prealigned_v2/` is generated by `anvil_exp01.data.prealign_v2`.
- `motion_labels.csv` is only required if you want per-motion-bin Vimeo summaries.

## Xiph 1080p

```text
data/xiph_1080p/
  sequences/
    crowd_run/
      00000_im1.png
      00000_im2.png
      00000_im3.png
  prealigned_v2/
    crowd_run/
      00000_im1.png
      00000_im3.png
```

Notes:

- The Xiph release subset should preserve the triplet IDs used by the evaluation script.
- If you also want MV Blend with basic prealignment, add `data/xiph_1080p/prealigned/`.
